→

Refresh Source
↓
Save Source
→
✅ snapshots/<plat>/
  home.xml ← from Inspector

你对 Claude 说: "dump 不出来,怎么办?"

技能做: 技能只是 Appium / UIAutomator2 / XCUITest driver 的薄包装,跟 Appium Inspector 用的是同一套。打开 [Appium Inspector](https://github.com/appium/appium-inspector),用你平时的 capabilities 连接,点 Refresh Source → Save Source → 得到 XML。放到 ~/.claude/skills/mobile-inspect/snapshots//.xml。技能的其他命令(--gen-pom、--merge、format-*、suggest-*、elements-summary)照常工作 — Inspector 和技能产出的 XML 可互换。

3. 每页输出 — 3 个并行文件

每次快照(或 --crawl-app 抓的每个页面)都写出3 个同名文件:

  • .xml — UI 树数据(--gen-pom 的输入)。
  • .png — 页面真实截图。事后回顾视觉上下文,或与 XML 对照。
  • .elements.md — 所有命名元素的 markdown 表:Name | Type | Bounds | Tap — 不解析 XML 也能快速读。

如果只要 XML,用 MOBILE_INSPECT_NO_EXTRAS=1 跳过另两个文件。

5. 完整工作流 — 构建 Page Object 库

用 4 条命令为整个应用构建 Page Object 库:

  1. 1️⃣ 打开 app,停在 Home — 游客模式
  2. 2️⃣ 自动爬取 — inspect.sh android --crawl-app(约 3-5 分钟)
  3. 3️⃣ 选模板 — inspect.sh --list-templates
  4. 4️⃣ 生成代码 + 写入项目 — inspect.sh android --gen-pom --template cross-platform-registry --target /path/to/wdio-project

结果:pages/*.page.ts + selectors/registries/*.ts 可直接粘贴,共享元素自动归入 BasePage。

⚠️ 安全

  • 用测试账号,绝对不要用生产账号。 技能真实点击。已登录账号上 → 可能误触 Like/Comment/Send/Delete/Pay。--crawl-app 默认游客模式 + 跳过危险关键词。
  • XML + PNG 可能包含 PII。 邮箱、姓名、搜索历史、消息预览。snapshots/ 已 gitignore,但仍按敏感数据对待。
