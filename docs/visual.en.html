📱 mobile-inspect — Visual Overview

Captures the live UI tree of Android/iOS apps so Claude can find Appium selectors, auto-crawl the whole app, and generate Page Objects. Each captured page produces 3 files: .xml (tree), .png (screenshot), .elements.md (element table).

You tell Claude: "can't dump, what now?"

Skill does: The skill is a thin wrapper around the same Appium / UIAutomator2 / XCUITest driver Appium Inspector talks to. Open [Appium Inspector](https://github.com/appium/appium-inspector), connect with your normal capabilities, click Refresh Source → Save Source → you get an XML. Drop it into ~/.claude/skills/mobile-inspect/snapshots//.xml. All other skill commands (--gen-pom, --merge, format-*, suggest-*, elements-summary) work as if the XML came from us — interchangeable.

3. Per-page output — 3 side-by-side files

Every snapshot (or each page captured by --crawl-app) writes 3 sibling files with the same basename:

  • .xml — UI tree data (consumed by --gen-pom).
  • .png — Real screenshot of the page. Look back at the visual context later, or compare against the XML.
  • .elements.md — Markdown table of every named element: Name | Type | Bounds | Tap — readable without parsing XML.

Skip the two extras with MOBILE_INSPECT_NO_EXTRAS=1 if you only need the XML.

5. Full workflow — build a Page Object library

Build a Page Object library for the whole app in 4 commands:

  1. 1️⃣ Open the app, stop at Home — guest mode
  2. 2️⃣ Auto-crawl — inspect.sh android --crawl-app (~3-5 min)
  3. 3️⃣ Pick a template — inspect.sh --list-templates
  4. 4️⃣ Generate + write to project — inspect.sh android --gen-pom --template cross-platform-registry --target /path/to/wdio-project

Result: pages/*.page.ts + selectors/registries/*.ts ready to paste, shared elements auto-promoted to BasePage.

⚠️ Safety

  • Use a test account, NEVER production. The skill taps real elements. On a logged-in account, taps can fire Like/Comment/Send/Delete/Pay unintentionally. --crawl-app defaults to guest mode + skips danger keywords.
  • XML + PNG dumps may contain PII. Emails, names, search history, message previews. snapshots/ is gitignored, but still treat as sensitive. Don't share, don't commit.
