2. Khi nào dùng?

🔍 Case 1 — Tìm tên element cụ thể

Tình huống: Đang viết test 'Click Like' nhưng không biết nút Like tên gì.

App on emulator
→

→
✅ Skill returns:

Skill làm: Chụp UI hiện tại → trả về ~LikeButton hoặc id/like_btn để copy vào test.

🐛 Case 2 — Test fail 'không thấy element'

Tình huống: Appium báo NoSuchElement nhưng nhìn thấy nút rõ ràng.

❌ Test code:
$("~SaveButton")
→ NoSuchElement
🔍
Skill finds:
name="SaveBtn"
→
✅ Fixed:
$("~SaveBtn")

Bạn bảo Claude: "inspect screen, sao Appium không thấy SaveButton?"

Skill làm: Chụp UI, chỉ ra element thật tên gì (vd SaveBtn không phải SaveButton) hoặc bị overlay che.

3. Output mỗi page — 3 file song song

Mỗi lần snapshot 1 page (hoặc khi --crawl-app tự chụp), skill ra 3 file song song cùng tên:

  • .xml — UI tree data (đầu vào cho --gen-pom)
  • .png — Ảnh chụp màn hình thật của page đó. Để bạn nhìn lại sau, hoặc so sánh visual với XML.
  • .elements.md — Bảng markdown liệt kê tất cả element có tên: Name | Type | Bounds | Tap — đọc nhanh không cần parse XML.

Skip 2 file extra bằng env MOBILE_INSPECT_NO_EXTRAS=1 nếu chỉ cần XML.

5. Workflow đầy đủ — sinh Page Object library

Dựng Page Object library cho cả app — 4 lệnh:

  1. 1️⃣ Mở app, dừng ở Home — guest mode
  2. 2️⃣ Auto-crawl — inspect.sh android --crawl-app (~3-5 phút)
  3. 3️⃣ Chọn template — inspect.sh --list-templates
  4. 4️⃣ Sinh code + ghi vào project — inspect.sh android --gen-pom --template cross-platform-registry --target /path/to/wdio-project

Kết quả: pages/*.page.ts + selectors/registries/*.ts paste-able, element shared tự promote vào BasePage.
