📱 mobile-inspect — 시각 개요

Android/iOS 앱의 실시간 UI 트리를 캡처합니다. Claude 가 Appium 셀렉터를 찾고, 앱 전체를 자동 크롤링하고, Page Object 를 직접 생성합니다. 캡처된 각 페이지마다 3 개 파일을 출력:.xml(트리)、.png(스크린샷)、.elements.md(요소 표).

Claude 에게 말하기: "dump 안 돼, 어떻게 해?"

스킬이 함: 스킬은 Appium / UIAutomator2 / XCUITest driver 의 얇은 래퍼 — Appium Inspector 와 같은 메커니즘. [Appium Inspector](https://github.com/appium/appium-inspector) 를 열고, 평소 capabilities 로 연결, Refresh Source → Save Source 클릭 → XML 획득. ~/.claude/skills/mobile-inspect/snapshots//.xml 에 배치. 스킬의 다른 명령(--gen-pom, --merge, format-*, suggest-*, elements-summary)은 그대로 작동 — Inspector 와 스킬의 XML 은 호환 가능.

3. 페이지별 출력 — 3 개 병렬 파일

각 스냅샷(또는 --crawl-app 이 캡처하는 각 페이지)마다 같은 베이스 이름의 3 개 형제 파일을 작성:

  • .xml — UI 트리 데이터(--gen-pom 의 입력).
  • .png — 페이지의 실제 스크린샷. 나중에 시각적 맥락을 다시 보거나, XML 과 대조.
  • .elements.md — 명명된 모든 요소의 마크다운 표:Name | Type | Bounds | Tap — XML 을 파싱하지 않고 빠르게 읽기.

XML 만 필요하면 MOBILE_INSPECT_NO_EXTRAS=1 로 두 추가 파일 스킵.

5. 전체 워크플로우 — Page Object 라이브러리 구축

앱 전체의 Page Object 라이브러리를 4 명령어로 구축:

  1. 1️⃣ 앱 열고 Home 에서 정지 — 게스트 모드
  2. 2️⃣ 자동 크롤 — inspect.sh android --crawl-app(약 3-5 분)
  3. 3️⃣ 템플릿 선택 — inspect.sh --list-templates
  4. 4️⃣ 생성 + 프로젝트에 쓰기 — inspect.sh android --gen-pom --template cross-platform-registry --target /path/to/wdio-project

결과:pages/*.page.ts + selectors/registries/*.ts 바로 붙여넣기 가능, 공유 요소는 자동으로 BasePage 로 승격.

⚠️ 안전

  • 테스트 계정 사용, 운영 계정 절대 금지. 스킬은 실제로 탭합니다. 로그인된 계정에서는 Like/Comment/Send/Delete/Pay 가 의도치 않게 발생 가능. --crawl-app 는 기본 게스트 모드 + 위험 키워드 스킵.
  • XML + PNG 에 PII 포함 가능. 이메일, 이름, 검색 기록, 메시지 미리보기. snapshots/ 는 gitignore 되어 있지만, 민감한 데이터로 취급.
